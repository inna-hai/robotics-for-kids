🎯 מטרות ותוצר

מושג מקור

משתנים הם “מגירות אחסון” ששומרות נתונים בזיכרון התוכנית.

סוגי מידע

מספרים שלמים, מספרים עשרוניים, טקסט, תווים וקלט מהמשתמש.

תוצר בלומדה

דמות player שמופיעה במסך לפי x ו־y.

שאלת מפתח

אם נשנה את x או y — מה יקרה לדמות?

📦 מה זה משתנה?

מתכנתים שומרים נתונים בזיכרון התוכנית באמצעות משתנים. אפשר לחשוב עליהם כמו מגירות אחסון.

שם המשתנה

שם משמעותי שמסביר מה שמרנו.

ערך

המידע שנמצא בתוך המשתנה.

שינוי ערך

אפשר להחליף ערך או לחשב ערך חדש.

שימוש במשחק

x, y, speed, score, playerName.

✍️ שמות משתנים — הסכמה

  • משתנים מתחילים באותיות קטנות.
  • אם יש יותר ממילה אחת, מחברים עם קו תחתון או אות גדולה.
  • כדאי לבחור שמות משמעותיים.
my_name = "Dana"
myName = "Dana"
points = 18
speed = 0.3
שאלת מדריך: איזה שם טוב יותר — x או player_x? תלוי בהקשר.

🔢 משתנים מספריים

משתנים מסוג מספר מאפשרים פעולות חשבוניות.

points = 18 + 1
x = 100 - 50
y = 20 * 8
speed = 0.3 / 10
  • חיבור +
  • חיסור -
  • כפל *
  • חילוק /

🔁 שינוי ערך קודם

במשחקים ערך משתנה כל הזמן: דמות זזה, ניקוד עולה, מהירות משתנה.

points = 5
points = points + 3

x = x + 5
x += 5

speed = speed - 0.1
חיבור ל־Pygame: כש־x גדל, הדמות זזה ימינה. כש־y גדל, היא זזה למטה.

🧪 תרגול מקור — גילאי משפחה

מגדירים גיל לכל בן בית, ואז מחשבים סכום, מכפלה וחילוק.

age1 = 18
age2 = 46
age3 = 44
age4 = 3

sumOfAges = age1 + age2 + age3 + age4
mulOfAges = age1 * age2 * age3 * age4
div = mulOfAges / sumOfAges

מטרת התרגול: להבין שמשתנים יכולים להשתתף בחישובים.

📝 מחרוזות ושרשור

מחרוזת היא טקסט בתוך מרכאות. אפשר לחבר מחרוזות עם +.

firstName = "James"
lastName = "Lacey"
fullName = firstName + " " + lastName

city = "tel aviv"
print(fullName + " from " + city)
שימו לב: טקסט בתוך מרכאות. שם משתנה בלי מרכאות.

⌨️ קלט מהמשתמש

כשרוצים לשאול את המשתמש שאלה, משתמשים ב־input ושומרים את התשובה במשתנה.

name = input("what is your name? ")
print("hello " + name)
  • הטקסט בתוך input הוא השאלה.
  • התשובה נשמרת בתוך name.
  • אפשר להשתמש בשם בהדפסה או במשחק.

🎮 מעבר ל־Pygame: משתנים כמיקום

במשחק, גם המיקום של הדמות הוא מידע שנשמר במשתנים.

player_x = 300
player_y = 220
player = pygame.Rect(player_x, player_y, 42, 42)
x

גדל ימינה.

y

גדל למטה.

width / height

גודל הדמות.

Rect

מלבן שמייצג דמות או אובייקט.

🧪 תרגול בלומדה — מזיזים דמות

  1. פתחו את שיעור 2 בלומדה.
  2. מצאו את בלוק player.
  3. שנו את x והריצו.
  4. שנו את y והריצו.
  5. כתבו: מה ההבדל בין שינוי x לשינוי y?
דגש מדריך: לפני ההרצה לשאול ניבוי — “לאן הדמות תעבור?”

🧪 אתגר — מרכז המסך

המסך שלנו הוא 640×480. הדמות היא בערך 42×42.

  1. נסו למקם את הדמות במרכז.
  2. בדקו אם היא באמת נראית באמצע.
  3. שנו קצת את x/y עד שהמיקום מרגיש נכון.
  4. הסבירו למה לא תמיד המרכז הוא פשוט 320,240.

⏱️ מבנה מומלץ — 90 דקות

  1. 0–10: חזרה: מה עשינו בשיעור 1?
  2. 10–25: הסבר משתנים כמגירות אחסון.
  3. 25–40: מספרים ופעולות חשבון.
  4. 40–52: תרגול גילאי משפחה.
  5. 52–65: מחרוזות, שרשור וקלט.
  6. 65–80: מעבר ל־Pygame: x/y ו־Rect.
  7. 80–90: הצגת מיקום דמות וסיכום.

🛠️ טעויות נפוצות ודיבאג

  • כותבים שם משתנה עם רווח.
  • שמים משתנה בתוך מרכאות ואז הוא מודפס כטקסט רגיל.
  • מחברים מספר וטקסט בלי str.
  • משנים גם x וגם y יחד ואז לא יודעים מה השפיע.
  • שוכחים שב־Pygame y גדל כלפי מטה.
